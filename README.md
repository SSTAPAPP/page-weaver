# FFk 理发店管理系统

专业的理发店会员管理与收银系统，涵盖会员开卡、储值充值、次卡消费、预约管理、收银结账、数据报表等完整业务闭环。面向中小型美发门店，提供开箱即用的一站式运营解决方案。

## 系统概览

FFk 是一套为理发店量身打造的 SaaS 管理系统，采用云端优先架构，支持多端自适应（桌面、平板、手机），核心业务数据实时同步至云端，确保数据安全与多设备一致性。

### 核心能力

| 能力 | 说明 |
|------|------|
| **会员体系** | 开卡、储值、次卡、标签分组、性别管理 |
| **智能收银** | 余额 / 次卡 / 现金 / 微信 / 支付宝多通道结账，自动补差价 |
| **预约调度** | 日历月视图、状态流转（待确认→已确认→已完成/未到/取消） |
| **数据洞察** | 营收趋势、充值消耗分析、会员增长统计 |
| **交易追溯** | 完整充值、消费、退款、作废记录链 |
| **安全管控** | 管理员密码保护、敏感操作二次验证、请求频率限制 |

## 功能模块

### 1. 仪表盘 `/`

实时掌握今日营收、会员动态与预约概况。

- 今日营收总额、消费笔数、新增会员数
- 待处理预约概览
- 关键业务指标卡片展示

### 2. 收银台 `/cashier`

选择会员与服务项目，支持多种支付方式快速结账。

- 会员搜索（支持姓名、手机号、拼音首字母）
- 服务项目选择与购物车管理
- 次卡自动匹配：系统自动检测会员次卡是否覆盖所选服务
- 多支付通道：余额扣减、次卡抵扣、现金、微信、支付宝
- 补差价逻辑：次卡不足时自动计算需补差额
- 散客模式：无需会员即可快速结账
- 结账小票生成与打印

### 3. 会员管理 `/members`

完整的会员生命周期管理。

- 新建会员：姓名、手机号、性别、标签（金卡/银卡/普通）
- 快速充值：直接为会员余额充值
- 次卡购买：从次卡模板中为会员开卡
- 会员搜索：支持姓名、手机号、拼音首字母模糊搜索
- 会员详情：余额、次卡、历史交易记录一览
- 会员删除：支持退款后删除，需管理员密码验证

### 4. 预约管理 `/appointments`

按日历查看与管理客户预约。

- 月视图日历展示，直观查看每日预约分布
- 新建预约：关联会员、选择服务、指定日期时间
- 预约状态流转：待确认 → 已确认 → 已完成 / 未到 / 已取消
- 预约详情查看与状态变更

### 5. 服务管理 `/services`

维护服务项目与次卡模板。

- **服务项目管理**
  - 添加/编辑/停用服务项目
  - 服务分类（如：剪发、染发、烫发、护理等）
  - 价格与时长设置
- **次卡模板管理**
  - 创建次卡模板：名称、价格、总次数
  - 关联适用服务项目
  - 模板启用/停用

### 6. 数据报表 `/reports`

营收趋势、充值消耗分析与会员增长统计。

- 营收趋势图表（按日/周/月）
- 充值与消费对比分析
- 会员增长曲线
- 各服务项目消费占比
- 支付方式分布统计

### 7. 交易流水 `/transactions`

完整的充值、消费、退款记录与作废操作。

- 交易类型筛选：充值 / 消费 / 次卡扣减 / 退款 / 补差价
- 交易详情查看：关联会员、支付方式、子交易明细
- 交易作废：需管理员密码验证，自动回滚会员余额/次卡
- 交易小票打印

### 8. 系统设置 `/settings`

店铺信息、主题切换、管理员密码与数据管理。

- 店铺基本信息：名称、地址、电话
- 主题切换：亮色 / 暗色模式
- 管理员密码管理：修改敏感操作验证密码
- 数据管理：数据导出与重置

## 技术架构

```
┌─────────────────────────────────────────────┐
│                  前端 (SPA)                  │
│  React 18 + TypeScript + Vite + Tailwind    │
│  Zustand (状态) + React Query (缓存/同步)    │
│  shadcn/ui (组件库) + Recharts (图表)        │
├─────────────────────────────────────────────┤
│              Lovable Cloud 后端              │
│  PostgreSQL 数据库 + Edge Functions          │
│  RLS 行级安全 + bcrypt 密码哈希              │
│  请求频率限制 + CORS 动态配置                │
└─────────────────────────────────────────────┘
```

### 前端技术栈

| 技术 | 用途 |
|------|------|
| **React 18** | UI 框架，函数组件 + Hooks |
| **TypeScript** | 类型安全，全量类型覆盖 |
| **Vite** | 构建工具，极速 HMR |
| **Tailwind CSS** | 原子化 CSS，设计令牌驱动 |
| **shadcn/ui** | 可定制组件库，基于 Radix UI |
| **Zustand** | 轻量状态管理（本地缓存层） |
| **@tanstack/react-query** | 服务端状态管理、缓存、自动重试 |
| **Recharts** | 数据可视化图表 |
| **date-fns** | 日期处理工具库 |
| **pinyin-pro** | 拼音搜索支持 |
| **React Router v6** | 客户端路由 |
| **Sonner** | Toast 通知 |

### 后端架构

- **数据库**：PostgreSQL，通过 Lovable Cloud 托管
- **Edge Functions**：Serverless 函数处理复杂业务逻辑（收银结账、密码验证等）
- **行级安全 (RLS)**：数据库层面的访问控制
- **实时同步**：通过 React Query 实现前端数据缓存与后台静默刷新

### 响应式设计

系统针对三种设备形态提供差异化布局：

- **桌面端** (`>1024px`)：侧边栏导航 + 宽内容区
- **平板端** (`768px-1024px`)：可折叠侧边栏 + 自适应内容
- **移动端** (`<768px`)：底部标签栏导航 + 更多菜单抽屉

## 数据模型

### 核心表结构

```
members          — 会员信息（姓名、手机、余额、性别、标签）
member_cards     — 会员次卡（关联模板、剩余次数、适用服务）
card_templates   — 次卡模板（名称、价格、总次数、关联服务）
services         — 服务项目（名称、价格、时长、分类）
appointments     — 预约记录（会员、服务、日期时间、状态）
transactions     — 交易流水（类型、金额、支付方式、子交易）
orders           — 结账订单（服务明细、支付明细）
shop_settings    — 店铺设置（名称、主题、管理员密码哈希）
audit_logs       — 审计日志（操作类别、详情、元数据）
rate_limits      — 频率限制（请求计数、时间窗口）
sync_queue       — 同步队列（离线操作缓冲）
```

### 关键关系

- `member_cards.member_id` → `members.id`（会员持有次卡）
- `appointments.member_id` → `members.id`（会员预约）
- `transactions.related_transaction_id` → 自引用（退款关联原交易）

## 安全机制

| 机制 | 说明 |
|------|------|
| **管理员密码** | bcrypt 哈希存储，支持从旧版 SHA-256 自动迁移 |
| **敏感操作验证** | 删除会员、交易作废等需输入管理员密码 |
| **请求频率限制** | 基于数据库的跨实例频率限制，防止暴力破解 |
| **服务端校验** | 收银结账等核心逻辑在 Edge Functions 中执行 |
| **RLS 策略** | 数据库行级安全策略控制数据访问权限 |
| **CORS 配置** | 动态 CORS，仅允许生产域名与授权预览环境 |

## 部署与发布

本项目采用 **GitHub Actions** 自动化 CI/CD 流程：

1. 代码通过 GitHub Desktop 推送至仓库
2. GitHub Actions 自动触发构建
3. 前端代码自动构建发布
4. Edge Functions 自动部署

- Node.js 18+

npm run dev

## 使用指南

### 快速上手

1. 打开系统，进入「服务管理」添加服务项目（如：男士剪发 ¥38）
2. 按需创建次卡模板（如：剪发10次卡 ¥300）
3. 在「会员管理」中添加会员
4. 为会员充值或购买次卡
5. 在「收银台」为会员结账消费

### 收银结账流程

1. 在收银台搜索并选择会员（或选择散客模式）
2. 添加服务项目到购物车
3. 系统自动检测可用次卡并匹配
4. 选择支付方式（余额/次卡/现金/微信/支付宝）
5. 确认结账，生成交易记录与小票

### 交易作废流程

1. 在「交易流水」中找到目标交易
2. 点击作废，输入管理员密码验证
3. 系统自动回滚：退还余额或恢复次卡次数
4. 生成对应的退款交易记录

## 常见问题

**Q: 次卡和余额可以混合使用吗？**
A: 可以。收银台支持次卡抵扣部分服务 + 余额/现金/微信/支付宝支付剩余差额。

**Q: 删除会员后数据还在吗？**
A: 删除会员时可选择退还余额，系统会生成退款记录。删除后会员数据不可恢复，但交易流水保留。

**Q: 如何修改管理员密码？**
A: 进入「系统设置」→ 管理员密码管理，输入旧密码验证后即可设置新密码。

**Q: 支持离线使用吗？**
A: 系统设计了同步队列机制，短暂断网时操作会缓存，恢复网络后自动同步。但建议保持网络连接以确保数据实时性。
